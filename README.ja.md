# snip2path

CLI（Claude Code等）での開発体験を爆速にするための、超軽量スニッピングツール。

[English version here](README.md)

## 解決したい課題

- Docker/WSL環境からWindowsのクリップボード画像を直接参照するのが困難。
- 画像を見せるために「保存 → パスをコピー → 伝える」という手順が発生し、思考が途切れる。

## 核心的な思想

- **「履歴は不要。最新の一枚があればいい」** — 画像は常に `latest.png` に上書き。
- **「パス管理を自動化」** — プロジェクトごとに `.snips/` フォルダを自動生成。
- **「視覚情報の維持」** — スニップ後、その画像をデスクトップに浮かせておく。

## 主要機能

1. **プロジェクト登録**
   - 指定したリポジトリ内に `.snips/` フォルダを自動作成。
   - `.snips/latest.png` を中央ストレージ `~/.snip2path/latest.png` にハードリンクで接続。

2. **スニッピング & 上書き保存**
   - 画面上で範囲を選択。
   - 常に `./.snips/latest.png` を上書き。バージョン管理不要。

3. **クリップボード連携**
   - 保存と同時に、相対パス `./.snips/latest.png` をクリップボードにコピー。

4. **フローティングプレビュー**
   - スニップした画像を浮きウィンドウとして表示。
   - 最前面固定にはせず、自由に配置・参照できる。
   - 次のスニップ時に自動で閉じる。

5. **キャプチャ遅延**
   - 3 / 5 / 7 秒の遅延を設定可能。画面を準備してからキャプチャできる。

## 技術スタック

- Python 3.10+
- PySide6 (GUI / オーバーレイ / ウィンドウ管理)
- Pillow (画像キャプチャ・処理)
- pyperclip (クリップボード)

## ビルド方法（Windows）

```bat
build.bat
```

`dist\snip2path\snip2path.exe` が生成されます。

## 動作環境

- Windows 10 / 11
- Python 3.10+（ソースからビルドする場合）
